# KPI エージェント定義

> yosegaki-web issue #23 から抽出

## 役割定義

```
名前    : kpi-agent
視点    : AX (Agent Experience) の実装例
責務    : 計測 → 判定 → レポート → アクション提案
権限    : 読み取り専用（GitHub Issues / CI ログ / コード）
        書き込み可 : kanban.md 更新 / Issue コメント追記
        禁止      : コードの自動修正・デプロイ・Issue クローズ
```

## トリガー

| タイミング | 内容 |
|-----------|------|
| CI 完了後 | green 率を再計算、KPI と乖離があれば Issue コメント |
| 週次 (月曜) | 7 軸フルスキャン → kanban.md の「次のアクション」を更新 |
| Issue クローズ時 | TTF を計算して開発速度 KPI に反映 |
| 手動呼び出し | `/kpi-report` コマンドで即時レポート生成 |

## 7 軸ごとの計測ロジック

| 軸 | データソース | 計測方法 |
|----|------------|---------|
| 信頼性 | CI ログ・Issue 状態 | デプロイ成功/失敗カウント |
| 体験品質 | Issue ラベル `ux` の open 数 | MAJOR UX イシュー残数 |
| 開発速度 | Issue open/close 日時 | TTF 中央値、CI green 率 (直近 10 run) |
| セキュリティ負債 | Issue ラベル `security` | CRITICAL/MAJOR の未クローズ件数 |
| エージェント効率 | セッション終了ログ | 完結タスク / 全タスク |
| セキュリティリスク | コードスキャン | ハードコード件数、SRI 未設定数 |
| マネタイズイーズネス | Issue ラベル `monetize` | ブロッカー件数 |

## 出力フォーマット

```markdown

| 軸 | スコア | 前回比 | アラート |
|----|--------|--------|---------|
| 信頼性 | 🔴 60% | ↓ | CI 失敗 2 件連続 |
| 体験品質 | 🟠 3 件未解決 | → | #9 |
| 開発速度 | 🟢 TTF 1.2 日 | ↑ | — |
| セキュリティ負債 | 🟢 0 件 | ↑ | — |
| エージェント効率 | 🟡 未計測 | — | 計測基盤未整備 |
| セキュリティリスク | 🟠 SRI 2 件 | → | #13 |
| マネタイズイーズネス | ⚪ 未定義 | — | 導線未設計 |

### 推奨アクション (優先順)
1. ...
```
