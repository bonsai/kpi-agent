# KPI ドリブン開発体制の構築

> yosegaki-web issue #22 から抽出

## 目的

DX / PX / AX の 3 視点を**数値で測れる状態**にする。

## 用語定義

| 用語 | 定義 |
|------|------|
| **信頼性 (Reliability)** | ユーザーが意図した操作が成功する割合。データロスゼロが前提。 |
| **体験品質 (UX Quality)** | 操作→フィードバックまでの明瞭さ。 |
| **開発速度 (Velocity)** | Issue open → close の平均日数。CI green 率も含む。 |
| **エージェント効率 (AX Score)** | 1 セッションで完結するタスクの割合。 |
| **セキュリティ負債** | CRITICAL / MAJOR イシューのうち未クローズのもの。常にゼロを目標。 |

## 目標数値

| KPI | 現状 | 目標 |
|-----|------|------|
| CI green 率 | ~60% | ≥ 95% |
| CRITICAL issue TTF | 未計測 | < 24h |
| MAJOR issue TTF | 未計測 | < 7 日 |
| データロス発生件数 | 未計測 | 0 件/月 |
| デプロイ成功率 | 0% (Secret 未設定) | 100% |
| AX Score (セッション完結率) | 未計測 | ≥ 80% |
